# Assignment 12 – Git & GitHub Commands

## 📖 Overview

This assignment demonstrates the practical use of **Git** and **GitHub** for version control. It covers the essential Git commands used to create repositories, track changes, manage commits, work with branches, and synchronize projects with GitHub.

## 🎯 Objectives

- Understand the basics of Version Control System (VCS).
- Learn to initialize and manage Git repositories.
- Practice staging and committing changes.
- Work with branches and merge workflows.
- Connect local repositories with GitHub.
- Push and pull changes using Git commands.

## 📂 Contents

This assignment includes Python programs demonstrating various Git-related concepts:

- Q01 – Repository Initialization
- Q02 – Git Configuration
- Q03 – Staging Files
- Q04 – Creating Commits
- Q05 – Viewing Commit History
- Q06 – Branch Management
- Q07 – Merging Branches
- Q08 – Working with Remote Repositories
- Q09 – Push & Pull Operations
- Q10 – Complete Git Workflow

## 🛠️ Technologies Used

- Git
- GitHub
- Visual Studio Code
- Windows PowerShell

## 🚀 Learning Outcomes

After completing this assignment, students will be able to:

- Create and manage Git repositories.
- Track project changes efficiently.
- Use Git commands for daily development.
- Collaborate using GitHub repositories.
- Maintain a clean and organized version history.
